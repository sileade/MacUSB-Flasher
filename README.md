# MacUSB Flasher

**MacUSB Flasher** — это универсальная консольная утилита для macOS (включая Apple Silicon M1/M2/M3/M4), предназначенная для создания загрузочных USB-накопителей.

Утилита автоматизирует процесс форматирования, разбивки и записи образов, избавляя от необходимости вручную вводить сложные команды в терминале.

## Возможности

| Режим | Метод записи | Поддерживаемые форматы |
|-------|-------------|------------------------|
| **Windows 10/11 (UEFI)** | FAT32 + GPT + wimlib split | `.iso` |
| **Linux** (Ubuntu, Debian, Mint, Arch...) | `dd` на `/dev/rdisk` | `.iso` |
| **Raspberry Pi / ARM** (Raspbian, DietPi...) | `dd` на `/dev/rdisk` | `.img`, `.img.gz`, `.img.xz`, `.zip` |
| **Универсальный режим** | `dd` на `/dev/rdisk` | любой `.iso` / `.img` |

### Ключевые особенности

**Автоопределение флешки** — скрипт сам обнаружит новую флешку при подключении, что исключает случайное форматирование системного диска.

**Автоустановка зависимостей** — при первом запуске самостоятельно установит `Homebrew`, `wimlib` и `xz`, если они отсутствуют в системе.

**Поддержка больших файлов Windows** — автоматически разбивает файл `install.wim` (если он больше 4 ГБ) для совместимости с файловой системой FAT32.

**Распаковка на лету** — поддерживает прямую запись сжатых образов (`.gz`, `.xz`, `.zip`).

**Максимальная скорость** — использует `/dev/rdisk` для ускорения побитовой записи.

**Верификация после записи** — после завершения записи скрипт автоматически проверяет целостность флешки: наличие загрузочных файлов (EFI, bootmgr, GRUB), читаемость файловой системы и корректность структуры. Для Linux/ARM-образов, где macOS не может прочитать ext4, проверяется успешность физической записи.

## Установка

Вы можете установить утилиту на рабочий стол для удобного запуска двойным кликом.

```bash
git clone https://github.com/sileade/MacUSB-Flasher.git
cd MacUSB-Flasher
chmod +x install_macusb_flasher.sh
./install_macusb_flasher.sh
```

На рабочем столе появится папка `MacUSB_Flasher`. Для запуска дважды кликните по файлу `MacUSB_Flasher.command`.

## Использование из терминала

Если вы не хотите устанавливать утилиту на рабочий стол, её можно запускать напрямую:

```bash
chmod +x MacUSB_Flasher.sh
./MacUSB_Flasher.sh
```

> **ВАЖНО:** Не запускайте скрипт через `sudo`! Скрипт сам запросит пароль администратора только в тот момент, когда это потребуется для форматирования диска.

## Обновление

Если утилита уже установлена и вы хотите обновить её до последней версии:

```bash
cd ~/Downloads/MacUSB-Flasher && git pull
cp MacUSB_Flasher.sh ~/Desktop/MacUSB_Flasher/
cp MacUSB_Flasher.command ~/Desktop/MacUSB_Flasher/
cp install_macusb_flasher.sh ~/Desktop/MacUSB_Flasher/
chmod +x ~/Desktop/MacUSB_Flasher/*.sh ~/Desktop/MacUSB_Flasher/*.command
```

## Как это работает

### Для Windows

macOS не поддерживает создание загрузочных флешек Windows через `dd`. Поэтому скрипт форматирует флешку в `MS-DOS (FAT32)` со схемой разделов `GPT`, монтирует ISO-образ, копирует все файлы через `rsync`, а если файл `sources/install.wim` превышает лимит FAT32 (4 ГБ), использует `wimlib-imagex` для его разделения на части (`.swm`). После записи проверяется наличие папки `EFI` и файла `bootmgr`.

### Для Linux и ARM

Используется классическая побитовая утилита `dd`. Для ускорения процесса запись идет на raw-устройство (`/dev/rdiskN` вместо `/dev/diskN`), а размер блока установлен в `4m`. После записи скрипт проверяет, завершилась ли операция без ошибок, и пытается найти загрузочные файлы на диске.

### Верификация

После каждой записи скрипт выполняет автоматическую проверку:

| Что проверяется | Windows | Linux/ARM |
|----------------|---------|-----------|
| Наличие EFI/boot папки | Да | Да (если доступно) |
| Наличие bootmgr | Да | Нет |
| Читаемость файловой системы | Да (FAT32) | Зависит от ФС |
| Успешность физической записи | Да | Да |
